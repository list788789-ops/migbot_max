## Известные риски / долги
- **Договор не отражает гарантии Крайнего Севера** (Мурманская обл. — район КС). Нет
  северной надбавки (ст. 317 ТК), нет доп. отпуска 24 к.д. (ст. 321 — отпуск должен быть
  52, стоит 28), нет класса СОУТ (ст. 57), нет вахтового характера. ОТЛОЖЕНО сознательно
  (пользователь: «пока убираем»). Обязанность работодателя по этим гарантиям сохраняется
  независимо от текста договора — работник вправе требовать. Показать юристу перед
  реальным оборотом. П.3.2 уже исправлен на безнал (валютный риск нерезидентов снят).
- **Пустой статус учёта блокирует создание обязательств** у всех 67 существующих (registration_status
  NULL после миграции). Кадровик обязан проставить статус каждому, иначе при заключении договора/
  подтверждении согласия обязательства (включая МВД/ЕФС-1) НЕ создадутся. Backfill в 'primary' НЕ
  делали по решению пользователя (пустой = не определено).
- **Заглушки-даты въезда** у не прибывших — кадровик обнуляет вручную (не прибывшие = пустой
  entry_date, тогда дедлайны не идут; заглушка даёт фантомные просрочки).
- **Двойные пробелы в ФИО** — почищены разово (regexp_replace \\s+ → ' '). Следить, чтобы
  форма ввода не плодила новые.

## Миграции БД (выполнить на боевой)
- ЭНАМ obligationtype в БД неполон относительно модели. Postgres не расширяет энам при
  добавлении значения в Python — нужен ALTER TYPE вручную. Добавлены: DACTYLOSCOPY;
  выполнить ещё: `ALTER TYPE obligationtype ADD VALUE 'EFS1_REPORT';` и
  `ALTER TYPE obligationtype ADD VALUE 'REGISTRATION_RENEWAL';`. При добавлении ЛЮБОГО
  нового типа обязательства — сразу ALTER TYPE на боевой, иначе DataError при первом
  запросе с этим типом.
- `ALTER TABLE employees ADD COLUMN registration_status VARCHAR;` (перед деплоем webforms).
